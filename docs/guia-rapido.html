Pré-requisitos

Antes de começar

1. Integração ativada no app

No repor, o mercado precisa ativar a integração em Ajustes > Automação Externa.

2. Chave de integração

O sistema externo vai autenticar usando x-integration-key.

3. Setores cadastrados no repor

Os packs precisam apontar para sectorId existente no repor.

4. Sistema com saída HTTP

O ERP, PDV ou sistema do mercado precisa conseguir fazer chamadas HTTP para a API do repor.

Fluxo

O que o sistema do mercado precisa enviar

1

Validar a chave

Fazer um GET /integrations/status para confirmar que a integração está pronta.

2

Enviar produtos

Sincronizar o catálogo por EAN usando POST /integrations/products/upsert.

3

Enviar packs / lotes

Sincronizar validade, quantidade e setor com POST /integrations/batches/sync.

4

Enviar vendas

Ao vender no caixa, enviar o evento para POST /integrations/sales para o repor dar baixa automática.

Resumo técnico

O que é importante alinhar com o cliente técnico

O repor envia dados manuais de volta para o ERP?

Não hoje. O fluxo atual é de entrada de dados do sistema externo no repor.

O modo manual continua funcionando?

Sim. Mesmo com a integração ativa, o mercado continua podendo operar manualmente no app.

O que identifica um produto?

O principal identificador é o EAN.

O que identifica um pack externo?

O principal identificador é o externalId do lote/pacote enviado pelo sistema externo.
