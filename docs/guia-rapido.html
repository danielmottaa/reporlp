Antes de começar

O que o integrador precisa ter em mãos

Checklist mínimo

  • Integração externa ativada no app do mercado.
  • x-api-key gerada pelo administrador.
  • JWT de administrador para consultar setores.
  • Setores já criados no repor.
  • Sistema com capacidade de fazer requisições HTTP.

Ponto importante

  • curl, terminal e Postman servem apenas para homologação.
  • A integração real roda no ERP, PDV, retaguarda ou backend do cliente.
  • O sistema precisa enviar EAN e quantidade ao repor.

Ambientes

Qual URL usar em cada fase

Homologação

Use https://api-hml.repor.app para testes, ajustes de payload e validação inicial.

Produção

Use https://api.repor.app apenas quando a integração já estiver validada.

Fluxo

O que o sistema do mercado precisa enviar

1

Consultar setores

Use GET /markets/current/integration/sectors com JWT admin para descobrir os sectorId.

2

Validar chave

Faça um GET /integrations/status.

3

Enviar produtos

Use POST /integrations/products/upsert.

4

Enviar packs

Use POST /integrations/batches/sync com sectorId válido.

5

Enviar vendas

Use POST /integrations/sales para baixa automática.

Entrega ao integrador

O que você deve enviar para o cliente técnico

1. Base URL correta

Homologação primeiro; produção só após aprovação.

2. Chave de integração

Entregue a x-api-key por canal seguro e apenas para uso no backend do integrador.

3. Setores do repor

Os packs precisam apontar para sectorId existente no repor. O caminho oficial é GET /markets/current/integration/sectors.

4. Documentação oficial

Abrir documentação técnica completa.
